{# ---------------- Command-line workflow ---------------- #}
Command line Edit in your own IDE, submit with cargo course
{% match ulid %} {% when Some with (u) %}

Prefer Zed, VS Code, or any other editor? The cargo course CLI lets you keep your local exercise files in sync with the same progress you see in the browser. Run the three steps below and you're set.

  1. 1. Set up the CLI

    Installs cargo-course and links this browser session to your machine in one paste. Safe to re-run — the install is a no-op if you already have the latest, and init just overwrites the saved token. 

    # 1. Install the CLI (one-time).
cargo install --git https://github.com/corrode/course cargo-course

# 2. Save your token so submissions land on this account.
#    Treat the token like a password — anyone with it can submit as you.
cargo course init --token {{ u }}

  2. 2. Submit a solution

    From the course directory, point submit at the exercise file you've been editing. Tests run locally first; only passing solutions are uploaded. 

    cargo course submit examples/00_integers/3_number_to_string.rs

    Add --pedantic to also run cargo fmt and cargo clippy and earn a ★ perfected badge. Use --all to bulk-submit every exercise that currently passes. Other handy commands:

    • cargo course status — show your progress in the terminal.
    • cargo course open — open your dashboard in the browser.
    • cargo course token — print the token currently saved on this machine.
